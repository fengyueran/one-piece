# Milestone v1.0: compass-ui 公开基线与复杂交互收口

**Status:** ✅ SHIPPED 2026-04-14
**Phases:** 1-4
**Total Plans:** 13

## Overview

`v1.0` 的目标不是继续横向扩张仓库范围，而是把 `@xinghunm/compass-ui` 收口成一套对外可安装、可阅读、可验证、可发布的 React 组件库主线。这个 milestone 先完成公开契约与发布基线，再统一文档与验证链路，补齐基础组件闭环，最后把复杂交互和 overlay 基础层收口。

## Phases

### Phase 1: 公开契约与发布基线

**Goal**: 外部消费者和仓库维护者都只依赖公开导出与单一发布基线使用和发布 `@xinghunm/compass-ui`
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] 01-01: 公开子路径、dumi alias 与文档示例收口到真实公开消费路径
- [x] 01-02: tarball 级别公开 API 验证与最小消费者夹具
- [x] 01-03: 根级发布前入口、Changesets access 与 Trusted Publishing workflow

**Delivered:**

- `compass-ui` 的首批公开子路径、dumi alias 和文档示例已全部收口到真实公开消费路径。
- `compass-ui` 已具备真实 tarball 级别的公开 API 验证脚本和只走公开导入路径的最小消费者夹具。
- `compass-ui` 的 access、peer、根级发布前入口和 Trusted Publishing workflow 已收口为一套可重复执行的发布基线。

### Phase 2: 文档入口与验证统一

**Goal**: 外部用户可以从单一入口理解并验证 `compass-ui`，且文档、demo、测试都复用同一条真实消费链路
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:

- [x] 02-01: dumi 单一文档入口与首页/API/组件目录信息架构
- [x] 02-02: 文档公开导入 smoke 脚本与根级验证入口
- [x] 02-03: 第一批键盘 / 可访问性自动化验证

**Delivered:**

- `compass-ui` 的 dumi 站点已收口为单一公开入口，首页、组件目录、安装指南与 API 页面形成统一导航。
- 文档与 demo 现在有了固定的公开导入 smoke 校验入口，维护者可以用一条命令同时验证 dumi build 和公开路径约束。
- 5 个高频组件已经补上第一批键盘与可访问性自动化断言，组件文档也同步声明了这些已验证行为。

### Phase 3: 基础组件补齐

**Goal**: 外部用户可以只依赖 `compass-ui` 完成常见表单输入、状态反馈、加载态与空态闭环
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:

- [x] 03-01: `checkbox / radio / switch`
- [x] 03-02: `Input / Textarea` 标准化与 `InputField` 兼容收口
- [x] 03-03: `Alert / Empty / Skeleton / SpinLoading`

**Delivered:**

- `compass-ui` 新增了可直接公开消费的 checkbox、radio、switch，并把布尔值字段接入了现有表单与文档体系。
- `compass-ui` 现在提供了标准化的 Input / Textarea，并把旧的 InputField 收口为兼容层与迁移入口。
- `compass-ui` 现在具备完整的页面状态基础层，外部用户可以直接使用 Alert、Empty、Skeleton、SpinLoading 覆盖持续反馈、空态和加载态。

### Phase 4: 复杂交互稳态化

**Goal**: 复杂选择与 overlay 组件具备一致的键盘、焦点和关闭行为，并形成可持续维护的基础层策略
**Depends on**: Phase 3
**Plans**: 4 plans

Plans:

- [x] 04-01: Select 键盘导航与 combobox/listbox 语义
- [x] 04-02: 现有 overlay 一致性收口
- [x] 04-03: `Popover / Popconfirm`
- [x] 04-04: `Drawer` 与复杂交互基础层策略

**Delivered:**

- Select 现在具备可完整使用的键盘导航、激活项跟踪与可验证的 combobox/listbox 对外语义。
- 现有 Dropdown、Tooltip、TreeSelect 与日期类浮层已经对齐到同一套 aria、关闭和内部点击保留约束。
- Popover 与 Popconfirm 已作为公开组件接入 Compass UI，并与现有 overlay 契约和文档边界保持一致。
- Drawer 已加入 Compass UI，并与 Modal 共用页面级 overlay 契约，同时仓库内补齐了复杂交互基础层策略说明。

## Milestone Summary

**Key Decisions:**

- 根 `package.json` 成为发布前校验和维护者流程的唯一权威入口。
- 文档、demo 与测试统一共享真实公开消费路径，不再依赖内部 alias。
- 基础组件优先于复杂交互；复杂交互完成后再明确基础层职责边界。
- 锚点型 overlay 继续基于 `Floating UI`，页面级 overlay 继续走 `Modal` / `Drawer` 的 portal + mask 基础层。

**Issues Resolved:**

- 补齐了 `compass-ui` 的公开子路径、pack 验证和消费者 smoke 检查。
- 统一了 dumi 文档站、API 页、组件目录和 docs smoke 验证。
- 补齐了高频基础组件与复杂交互组件的公开门面、文档和测试。
- 修复了 `Alert`、`Empty`、`Skeleton` 的 `title` 类型与 `HTMLAttributes` 冲突，确保 DTS 构建通过。

**Issues Deferred:**

- `.dumirc.ts` 尚未加入 `tsconfig.json#include`
- `Browserslist` 数据过旧

**Technical Debt Incurred:**

- `publint` 仍提示 `exports["."].types` 歧义
- package `"type"` 字段建议尚未处理
